# Wire contracts (frozen)

Everything the backend emits and the frontend consumes. The Python source of
truth is [`backend/app/models.py`](../backend/app/models.py); this document is
the human-readable mirror plus the HTTP surface.

> ⚠️ Treat this file as **frozen**. The FastAPI SSE stream, the CLI renderer,
> the React frontend, and the recorded-run demo fixtures all depend on it.

---

## 1. `PipelineEvent` — the streaming unit

Every step of the pipeline emits one `PipelineEvent`. It is the `data:` payload
of each SSE frame **and** the element type of the recorded-run fixtures.

```jsonc
{
  "seq": 0,                     // monotonic counter within a run, starts at 0
  "event": "run_started",       // see EventType below
  "stage": null,                // see Stage below, or null for run-level events
  "attempt": 1,                 // re-pick attempt this event belongs to (1-based)
  "elapsed_ms": 0,              // ms since run start — drives demo replay timing
  "message": "Matching @art_girl…",
  "data": null                  // stage payload (object | array | null) — see §3
}
```

### `EventType`
| value | meaning | `data` payload |
|---|---|---|
| `run_started` | pipeline began | `{ "username": "@art_girl", "mode": "mock", "user_platforms": ["netflix","hbo"] }` |
| `stage_started` | a stage began | `null` |
| `stage_progress` | mid-stage note (e.g. a tool call) | `{ "note": "…" }` (optional) |
| `stage_completed` | a stage finished | the stage's output — see §3 |
| `stage_failed` | a stage errored | `{ "error": "…" }` |
| `repick` | a pick wasn't on the user's subs → re-picking | `{ "excluded": [{ "title": "Fleabag", "reason": "only on prime" }] }` |
| `run_completed` | done | the full `RunResult` (§2) |
| `run_failed` | fatal error | `{ "error": "…" }` |

### `Stage`
`insta_reader` · `interest_profiler` · `show_matcher` · `streaming_checker`
(Human labels in `STAGE_LABELS`; canonical order in `STAGE_ORDER`.)

## 2. `RunResult` — the final payload (`run_completed.data`)

```jsonc
{
  "username": "@art_girl",
  "mode": "mock",                       // "real" | "mock"
  "profile": {                          // InterestProfile (stage 2 output)
    "primary_interests": ["analog film photography", "indie & arthouse cinema", "…"],
    "aesthetic_vibe": "soft grainy film-still energy — vintage, creative, intentional",
    "recommended_genres": ["prestige drama", "dark comedy", "romance"]
  },
  "picks": [                            // 0–3 FinalPick, ranked best-first (0 if the
                                        // matcher ran out of options on your subs)
    {
      "title": "The White Lotus",
      "year": 2021,
      "genres": ["dark comedy", "drama", "mystery"],
      "why": "Sharp, gorgeous and endlessly discussable — matches her arthouse, …",
      "platforms": ["hbo"]              // user-subscribed platforms it streams on
    }
  ],
  "excluded": [                         // titles dropped for not being on the subs
    { "title": "Fleabag", "reason": "only available on prime (not subscribed)" }
  ],
  "attempts": 2,                        // how many matcher attempts it took
  "user_platforms": ["netflix", "hbo"]
}
```

## 3. Per-stage `stage_completed.data`

| stage | `data` shape |
|---|---|
| `insta_reader` | `ProfileDump`: `{ "handle", "display_name", "bio", "posts": [], "hashtags": [] }` |
| `interest_profiler` | `InterestProfile` (see §2 `profile`) |
| `show_matcher` | `{ "candidates": [{ "title", "year", "genres": [], "why" }] }` (this attempt's picks) |
| `streaming_checker` | `{ "results": [{ "title", "found", "available", "platforms": [], "all_platforms": [] }] }` |

## 4. HTTP surface (FastAPI)

| method | path | purpose | response |
|---|---|---|---|
| `GET` | `/health` and `/api/health` | liveness + mode | `{ "status":"ok", "version":"1.0.0", "mode":"mock", "model_analysis":"…", "model_fast":"…", "user_platforms":["netflix","hbo"] }` |
| `GET` | `/api/profiles` | known demo handles for the picker | `[{ "handle":"@art_girl", "display_name":"mara · 35mm" }]` |
| `GET` | `/api/stream?handle=@art_girl` | **SSE** live pipeline | `text/event-stream`, each frame's `data:` is a `PipelineEvent` (default `message` event). Stream closes after `run_completed`/`run_failed`. |
| `POST` | `/api/run` | one-shot, non-streaming | body `{ "handle":"@art_girl" }` → `RunResult` |

CORS allows the Vite origins in `CORS_ORIGINS`. SSE clients should use native
`EventSource` (GET, auto-reconnect) and `es.close()` on `run_completed`.

## 5. Recorded-run demo fixtures (frontend demo mode)

When no backend is reachable (e.g. on Cloudflare Pages), the frontend replays a
bundled recording instead of streaming live. One file per known handle lives in
`frontend/src/demo/` and is generated by `datenight demo-fixtures` from a real
**mock-mode** run, so it is byte-for-byte what the live stream would produce.

```jsonc
// frontend/src/demo/art_girl.json
{
  "username": "@art_girl",
  "mode": "mock",
  "events": [ /* PipelineEvent[] in order, each with elapsed_ms */ ]
}
```

**Replay rule:** walk `events`, scheduling each by its `elapsed_ms` (optionally
time-compressed by a constant factor for a snappier demo). This reproduces the
exact live experience — staged reveal, the re-pick on `@art_girl`/`@tech_babe`,
and the final cards — with zero backend.
